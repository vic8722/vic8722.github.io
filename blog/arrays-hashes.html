Arrays and Hashses

March 6, 2016

Arrays

Arrays are useful for collections of data, particularly those that need to be in a certain order. The values stored in an array can be accessed by their index number, starting with the first value stored at index 0.

So for example, here is how you would use Ruby to create an array of random numbers:

my_array = [1,2,7,4,15]
=> [1, 2, 7, 4, 15]

Pretty neat, huh?

Let's say you are organizing a marathon and are tasked with keeping track of what order the finishers complete the race. You can use an array for that!

First, before the race begins we can set up an empty array:

finishers = Array.new
      => []

Now that we have an array to hold our names, we can add each person as they come across the finish line.

finishers << "Sarah"
      => ["Sarah"]
finishers << "Greg"
      => ["Sarah", "Greg"]
finishers << "Alice"
      => ["Sarah", "Greg", "Alice"]

And so on. One of the benefits in writing the code this way using an array is it keeps the values in the same order. This way, if you want to know who came in first place you can access the name in index 0:

finishers[0]
      => "Sarah"

Or, if you want to know who came in last, you can reference the numbers starting from the end using negative numbers. To find out who came in last you can use -1.

finishers[-1]
      => "Alice"

Hashes

Hashes are similar to arrays, however instead of being collections stored in numerical order they are collections of key and value pairs.

Hashes can be particularly useful when you want to keep track of data as it relates to something else. For example, let's say you wanted to know the bib number for each person running in the marathon. You can make a hash for that!

bibs = {"Alice" => 4827, "Greg" => 1337, "Sarah" => 8869}
=> {"Alice"=>4827, "Greg"=>1337, "Sarah"=>8869}

Now, if we want to know what Sarah's bib number is after she won the race we can look it up much like we look up values in the array, only instead of using an index we will use the key.

bibs["Sarah"]
      => 8869
