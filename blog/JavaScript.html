Ruby Hashes vs. JavaScript Objects

March 27, 2016

Ruby Hashes and JavaScript Objects are very similar, but do have a few differences. Let's start out by looking at how we create them in each language.

Starting off with Ruby. To create a new hash in ruby we can do this using bracket notation.


        ages = {"Tom" => 25, "Bob" => 31, "Jane" => 28}

        => {"Tom"=>25, "Bob"=>31, "Jane"=>28}

Similarly, we can write this same hash using the new method.


        ages = Hash.new
        ages["Tom"] = 25
        ages["Bob"] = 31
        ages["Jane"] = 28

        puts(ages)
        => {"Tom"=>25, "Bob"=>31, "Jane"=>28}

JavaScript also has two ways you can create new objects. Like Ruby, you can use bracket notation:


        var ages = {tom: 25, bob: 31, jane: 28};

        console.log(ages);
        => { tom: 25, bob: 31, jane: 28 }

Or, you can use the new keyword to make the same object:


        var ages = new Object();
        ages.tom = 25;
        ages.bob = 31;
        ages.jane = 28;

        console.log(ages);
        => { tom: 25, bob: 31, jane: 28 }

One difference, however, is in how you call elements from hashes and objects in each language. In Ruby you need to use square brackets to call each key to return the value:


        puts(ages["Tom"])
        => 25

In Javascript, you can do this using brackets like in Ruby:


        console.log(ages["tom"])
        => 25

But you can also do the same thing using dot notation:


        console.log(ages.tom)
        => 25

As you can see, both Ruby Hashes and JavaScript Objects are very similar.
